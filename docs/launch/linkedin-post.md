## Long Version

I’ve been building **Mnemon**, a brain-inspired memory framework for AI agents.

The idea is simple: most assistants still behave like stateless chat with a thin retrieval layer. I wanted something closer to a real long-running companion:

- episodic memory for experiences
- semantic memory for distilled facts
- procedural memory for skills and workflows
- working memory for active goals and current context
- a daemon layer that can browse, inspect files, edit the workspace, and keep operating over time

It now includes a Jarvis-style daemon with:

- persistent goals
- memory-aware chat
- web UI
- MCP bridge
- browser + workspace tooling from plain language
- grounded action memory, so verified tool use becomes future context

The repo is here:

`https://github.com/rohit27-02/mnemon`

If you’re working on agent memory, long-running assistants, or tool-using local agents, I’d love feedback.
